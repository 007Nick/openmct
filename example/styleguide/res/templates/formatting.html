Open MCT Style Guide

Formatting

ID and Class Naming

  • ID and class names should be human-readable: use names that are as short as possible while long enough to be meaningful. Ideally, names are semantic to the purpose of the element. Avoid names that don't convey meaning.
  • Avoid class names that refer overtly to the presentation of an object (e.g. .red) - this strategy is brittle and doesn't allow for themeing.
  • Generic names may be used for classes that serve as "helpers" such as .alt.

Bad

/* Not recommended, meaningless */ .eg-991 {} /* Not recommended, presentational */ .red {} .left {}

Good

/* Recommended, meaningful */ .form-row {} .form-section {} .plot-axis-x {} /* Recommended, specific */ .limit-lower {} .limit-upper {} /* Recommended, generic */ .align-right {} .ellipsis {}
