Tables

Simple Table

Tables in Open MCT are used to display data in a rows and columns format. There's no need to add any CSS classes to markup to effect the proper look, but be sure to use <th>'s in the <thead> section to get the proper look.

Sortable Table

Table headers typically enable users to sort the table by a given column either ascending or descending. Clicking on the header sorts by that column, first ascending, then toggling to descending. Sometimes, particularly with data, there is a use case to display and maintain the "original" no-sorted order: however the data arrives, display it in that order. In that case, we recommend that clicking a header repeatedly would first sort ascending, then descending, then back to the original order - no sort.

Note that this table illustrates sorting by Telemetry ID ascending and isn't functional: clicking the headers won't do anything.

Use the following CSS classes: add sortable to each <th> that is capable of sorting. The column that is currently driving the sort order should also be given the sort class. Finally, the sort direction is indicated with the addition of asc or desc. For example, a column that is currently sorting descending would have these classes attached to it: sortable sort desc.

Attach click handlers to the entire <th> element (not just to a link within) in order to maximize the usable click area in the header.

Table with Filters

Table styles also support the addition of column filters. Simply add filterable to the table's class attribute, and add text or number inputs to a row in the <thead> for each column that is filterable. Filtering should take place for each key press in a given field.
